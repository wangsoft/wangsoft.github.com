产品经理解构

1.能力要求

1.1 学习思考能力

不断地尝试，不断地学习，不断地认知，完成产品更准确的认知

1.2 执行力

对公司，对政策，对领导任务的执行力，完成预定目标和任务的能力

1.3 精神意志力：心态和情商（AQ&EQ）

积极主动面对困难及压力，以开放的心态迎接变化和挑战，并推动问题的最终解决。

1.4 沟通能力

1.4.1 基础沟通能力

  • 与各种周边角色沟通协作
  • 语言沟通能力
  • 书面沟通能力

1.4.2 商务沟通能力

理解合作方的利益点及自己可提供的资源，通过一定的谈判技巧，形成共赢的成交方案。

1.5 行业市场

1.5.1 行业融入

了解公司所属行业的上下游产业链，政策环境，最新的行业动态等。个人的产品项目在行业处于领先水平，有较高的知名度，同时个人能力为行业认知，有一定的影响力。

1.5.2 渠道管理

开拓和维护用户或内容的来源渠道，优化渠道结构，形成渠道合力并规避风险，建设利于产品发展的优质渠道体系。

1.5.3 市场分析能力

对行业情报、竞争对手动态和用户变化进行掌握和分析，确定产品的市场地位，掌握竞争格局，预测市场变化，确定战略战术。

1.5.4 市场营销

根据目标用户、产品特点及品牌塑造需要，进行营销及公关策略的制定和执行，以实现有效传播、危机化解、产品目标达成。

1.5.5 市场/用户的调研与分析

主动通过各种渠道了解用户反馈，掌握定性、定量调研方法，并将用研方法付诸于产品实践，持续优化产品。

1.6 基础知识能力

1.6.1 技术基础修养

了解与产品相关的技术实现原理及其表现形式，能够就技术方案与技术人员有效沟通，具备技术实现的基本技术概念。

1.6.2 关联知识

心理学、美学基础、财务知识、办公技能等

1.7 项目管理

通过流程规划、时程安排、任务和人员的管理以及资源的整合运用，顺利达成项目目标。

1.8 规划设计分析

1.8.1 产品规划，版本发布

准确把握用户需求，进行优先级排序，明确版本规划，通过迭代实现产品目标。明确业务逻辑，协助业务方制定业务规划和配套产品规划，推进产品线业务目标实施。

1.8.2 数据分析

通过设计数据指标体系，进行数据的收集和分析，挖掘潜在规律和问题，以优化产品和支撑决策。

1.8.3 专业设计能力

依据用户使用场景，使用相关专业领域的知识、工具和技巧，设计出满足甚至超出用户预期的功能特性。

1.9 知识积累和分享

1.9.1 方法论构建

在工作中主动帮助他人提升专业能力或者提供发展机会，帮助他人的学习与进步。不只是自己学习，更强调的是把经验的积累上升到方法论水平，提升团队整体能力。

1.9.2 知识传承

从工作积累中不断总结提炼，形成普遍性解决方案，起到指导及示范性作用，并加以推广应用。主动将自己所掌握的知识信息、资源信息，能通过交流、培训等形式分享，推动团队共同提高。

2. 岗位的要求

2.1 产品概念阶段

  • 在公司内外寻找产品创意。组织进行论证和充实。
  • 组织所辖产品线的市场细分选择，并制定产品线初始业务/路标计划（需求规格/RoadMap）。
  • 根据市场变化进行定期和不定期的计划调整工作。
  • 参与产品战略和产品平台规划工作。

2.2 产品需求阶段

  • 组织所辖产品的需求采集。
  • 组织收集/分析宏观环境，技术趋势，竞争对手，内外部客户的信息。
  • 组织对于产品相关的各种战略，计划，策略的审计工作。
  • 研究市场动态，提交市场研究报告，选择细分市场，确定产品定位。
  • 收集各个部门第产品初始业务/路标的意见。

2.3 产品设计阶段

  • 组织完成从产品创意到产品设计，形成完整的产品业务需求。
  • 组织对产品设计的测试工作。
  • 提交完成的产品业务需求，协调相关资源。
  • 提交产品开发任务书，获得授权，督导产品开发工作。

2.4 产品开发阶段

  • 监督产品开发计划，产品业务需求的完成情况。
  • 组织产品的市场调研工作，收集产品信息，根据需要调整产品业务需求和产品开发计划。
  • 组织或者参与研发开发阶段评审。
  • 协调资源对产品开发过程中的中间交付件进行测试。
  • 指导产品开发过程。

2.5 产品测试阶段

  • 组织产品的测试工作。
  • 制定产品的上市计划，为产品上市做培训，文档等前期准备工作。

2.6 产品发布阶段

  • 负责产品的市场发布工作。
  • 指导并监督产品的运营和销售工作。
  • 协同财务/市场部门监控产品的盈利情况，提出新的营销策略。
  • 根据市场反馈，提出产品的改进意见/监督执行

3 三个层级

3.1 初级:推动产品落地（怎么做）

  • 目的：需求的落地实现（按时保质完成任务）
  • 主要工作：信息传递和沟通
    • 了解交互
    • 了解设计
    • 了解研发内容
  • 工作细项:
    • 竞品分析
    • 原型设计
    • 日报总结
    • 写邮件
    • 方法论探索

3.2 中级: 引导需求（怎么做，做什么）

  • 目的：需求的挖掘和分析（通过资源调动，优化产品，完成KPI）
  • 主要工作：清晰取舍，引导客户，说服领导
  • 工作细项:
    • 数据分析
    • 用户研究
    • 需求挖掘
    • 需求评审
    • 产品设计
    • 组织协调
    • 资源调动
    • 项目推进
    • 方法论初成

3.3 高级: 战略规划（做什么，为什么，怎么做）

  • 目的：产品引导到正确方向（定方向）
  • 主要工作：保证每份力都用在公司的同一个正确方向
  • 工作细项:
    • 定方向
    • 项目管理
    • 团队管理
    • 时间管理
    • 品牌管理
    • 方法论传承
