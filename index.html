彻底删除趋势的 OfficeScan

  1. 开始->执行->msconfig，把所有有关office scan的服务 + 启动项目全部取消勾选
  2. 重开机进入安全模式
  3. 开启regedit，在左面的窗口中找到如下路径
    HKEY_LOCAL_MACHINE\SOFTWARE\TrendMicro\PC-cillinNTCorp\CurrentVersion\Misc.
    如果是64为系统的话，路径是
    HKEY_LOCAL_MACHINE\SOFTWARE\Wow6432Node\TrendMicro\PC-cillinNTCorp\CurrentVersion\Misc.，把Allow Uninstall值修改为1，NoPwdProtect值修改为1
  4. 欢乐卸载

产品经理解构

1.能力要求

1.1 学习思考能力

不断地尝试，不断地学习，不断地认知，完成产品更准确的认知

1.2 执行力

对公司，对政策，对领导任务的执行力，完成预定目标和任务的能力

1.3 精神意志力：心态和情商（AQ&EQ）

积极主动面对困难及压力，以开放的心态迎接变化和挑战，并推动问题的最终解决。

1.4 沟通能力

1.4.1 基础沟通能力

  • 与各种周边角色沟通协作
  • 语言沟通能力
  • 书面沟通能力

1.4.2 商务沟通能力

理解合作方的利益点及自己可提供的资源，通过一定的谈判技巧，形成共赢的成交方案。

1.5 行业市场

1.5.1 行业融入

了解公司所属行业的上下游产业链，政策环境，最新的行业动态等。个人的产品项目在行业处于领先水平，有较高的知名度，同时个人能力为行业认知，有一定的影响力。

1.5.2 渠道管理

开拓和维护用户或内容的来源渠道，优化渠道结构，形成渠道合力并规避风险，建设利于产品发展的优质渠道体系。

1.5.3 市场分析能力

对行业情报、竞争对手动态和用户变化进行掌握和分析，确定产品的市场地位，掌握竞争格局，预测市场变化，确定战略战术。

1.5.4 市场营销

根据目标用户、产品特点及品牌塑造需要，进行营销及公关策略的制定和执行，以实现有效传播、危机化解、产品目标达成。

1.5.5 市场/用户的调研与分析

主动通过各种渠道了解用户反馈，掌握定性、定量调研方法，并将用研方法付诸于产品实践，持续优化产品。

1.6 基础知识能力

1.6.1 技术基础修养

了解与产品相关的技术实现原理及其表现形式，能够就技术方案与技术人员有效沟通，具备技术实现的基本技术概念。

1.6.2 关联知识

心理学、美学基础、财务知识、办公技能等

1.7 项目管理

通过流程规划、时程安排、任务和人员的管理以及资源的整合运用，顺利达成项目目标。

1.8 规划设计分析

1.8.1 产品规划，版本发布

准确把握用户需求，进行优先级排序，明确版本规划，通过迭代实现产品目标。明确业务逻辑，协助业务方制定业务规划和配套产品规划，推进产品线业务目标实施。

1.8.2 数据分析

通过设计数据指标体系，进行数据的收集和分析，挖掘潜在规律和问题，以优化产品和支撑决策。

1.8.3 专业设计能力

依据用户使用场景，使用相关专业领域的知识、工具和技巧，设计出满足甚至超出用户预期的功能特性。

1.9 知识积累和分享

1.9.1 方法论构建

在工作中主动帮助他人提升专业能力或者提供发展机会，帮助他人的学习与进步。不只是自己学习，更强调的是把经验的积累上升到方法论水平，提升团队整体能力。

1.9.2 知识传承

从工作积累中不断总结提炼，形成普遍性解决方案，起到指导及示范性作用，并加以推广应用。主动将自己所掌握的知识信息、资源信息，能通过交流、培训等形式分享，推动团队共同提高。

2. 岗位的要求

2.1 产品概念阶段

  • 在公司内外寻找产品创意。组织进行论证和充实。
  • 组织所辖产品线的市场细分选择，并制定产品线初始业务/路标计划（需求规格/RoadMap）。
  • 根据市场变化进行定期和不定期的计划调整工作。
  • 参与产品战略和产品平台规划工作。

2.2 产品需求阶段

  • 组织所辖产品的需求采集。
  • 组织收集/分析宏观环境，技术趋势，竞争对手，内外部客户的信息。
  • 组织对于产品相关的各种战略，计划，策略的审计工作。
  • 研究市场动态，提交市场研究报告，选择细分市场，确定产品定位。
  • 收集各个部门第产品初始业务/路标的意见。

2.3 产品设计阶段

  • 组织完成从产品创意到产品设计，形成完整的产品业务需求。
  • 组织对产品设计的测试工作。
  • 提交完成的产品业务需求，协调相关资源。
  • 提交产品开发任务书，获得授权，督导产品开发工作。

2.4 产品开发阶段

  • 监督产品开发计划，产品业务需求的完成情况。
  • 组织产品的市场调研工作，收集产品信息，根据需要调整产品业务需求和产品开发计划。
  • 组织或者参与研发开发阶段评审。
  • 协调资源对产品开发过程中的中间交付件进行测试。
  • 指导产品开发过程。

2.5 产品测试阶段

  • 组织产品的测试工作。
  • 制定产品的上市计划，为产品上市做培训，文档等前期准备工作。

2.6 产品发布阶段

  • 负责产品的市场发布工作。
  • 指导并监督产品的运营和销售工作。
  • 协同财务/市场部门监控产品的盈利情况，提出新的营销策略。
  • 根据市场反馈，提出产品的改进意见/监督执行

3 三个层级

3.1 初级:推动产品落地（怎么做）

  • 目的：需求的落地实现（按时保质完成任务）
  • 主要工作：信息传递和沟通
    • 了解交互
    • 了解设计
    • 了解研发内容
  • 工作细项:
    • 竞品分析
    • 原型设计
    • 日报总结
    • 写邮件
    • 方法论探索

3.2 中级: 引导需求（怎么做，做什么）

  • 目的：需求的挖掘和分析（通过资源调动，优化产品，完成KPI）
  • 主要工作：清晰取舍，引导客户，说服领导
  • 工作细项:
    • 数据分析
    • 用户研究
    • 需求挖掘
    • 需求评审
    • 产品设计
    • 组织协调
    • 资源调动
    • 项目推进
    • 方法论初成

3.3 高级: 战略规划（做什么，为什么，怎么做）

  • 目的：产品引导到正确方向（定方向）
  • 主要工作：保证每份力都用在公司的同一个正确方向
  • 工作细项:
    • 定方向
    • 项目管理
    • 团队管理
    • 时间管理
    • 品牌管理
    • 方法论传承

ZeroNet

ZeroNet是一个利用比特币加密和BT技术提供不受审查的网络与通信的BT平台，ZeroNet网络功能已经得到完整的种子的支持和加密连接，保证用户通信和文件共享的安全。使用ZeroNet，你可以匿名上网，你可以在自己电脑搭建网站，但即使你关机，你的网站依然在全球存在，别人无法关闭你的网站。

官网：https://zeronet.io/
代码：https://github.com/HelloZeroNet/ZeroNet

什么是ZeroNet?

ZeroNet是一个去中心化的类似于Internet的网络，由匈牙利的开发者使用Python制作，完全开源。网站由特殊的”ZeroNet URL”可以被使用一般的浏览器通过ZeroNet程序浏览，就像访问本地主机一样。ZeroNet默认并「不」匿名，但是用户可以通过内置的Tor功能进行匿名化。ZeroNet使用Bitcoin加密算法及BitTorrent网络。（翻译于ZeroNet的维基百科页面）

防DMCA Take down

由于ZeroNet去中心化的原因,国外有人制作了一个ZeroNet版本的海盗湾——Play,这个网站可让版权所有者头疼啦…一般的网站服务器都有IP地址,不抗投诉的ISP收到了向IP WHOIS滥用信箱发送的DMCA投诉信的话,你就要和你的网站说再见了..(俗称 DMCA Take down),可用了ZeroNet后,并没有一台服务器是真实的服务器,任何访客都有可能成为服务器,还有些访客使用了Tor,更找不到真正的IP地址了,你说这DMCA信该向谁发呢?

Resilio Sync

Resilio Sync 是什么？

Resilio Sync (以前叫做 BitTorrent Sync) 是一款颠覆性的免费跨平台文件同步软件！它不需第三方服务器即可让多台电脑之间直接互相同步和共享文件，而且全程AES加密。你可以将 Resilio Sync 看作是一个没有流量和空间限制的私有个人化的 Dropbox。它采用基于类似BT下载的P2P分布式技术而来，速度快而且可通过密钥文件共享，是现今流行的云存储服务外又一颠覆性新选择……

Resilio Sync 的优点？

  • “同步工具”优势
  1. 不需要有自己的服务器
  2. 不需要有公网 IP和80权限，只要两台电脑都能访问到公网，就可以相互同步
  3. 文件数量无限制，文件大小无限制，空间只受制于你自己的硬盘
  4. 支持多种网络形态，可以“公网上互相同步”，也可以是“局域网内相互同步”，会自动识别出来
  5. 全平台支持
  • “分布式网盘工具”的优势
  1. 不受限于空间，dropbox和百度云都会受限，这个不会
  2. 没有流量限制和分享数量限制，这个完全看电脑自身和所在网络了
  3. 单个文件大小也不限制，反正是你自己说了算
  4. 分享也没有限制，没有审查，你自己说了算。不过并不代表可以做不和谐的事情，大家还是要注意分寸
  5. 本身没有费用，付费版也只是一些高级功能而已，屌丝们不用头疼了
  6. 不会被轻易KILL，这个工具本身是合法且有魅力的，封杀它实在是很不合理
  • “安全性”优势
    不同的 Resilio Sync 节点之间进行数据传输时，会采用“强加密”的方式，以防止数据传输流量被嗅探（偷窥）。
    只有参与同步的节点，才能解密；而那些帮你中转的“中转服务器”，是没有办法解密的。因此，即使你的 ISP（电信运营商）监视你的流量，也【无法】知道你通过 Resilio Sync 传输了啥文件。

Resilio Sync 的不足？

Resilio Sync 的不足在于拥有文件的电脑需要保持在线，才能同步到其他电脑去。而且由于采用类似 BT 的协议，因此在某些禁用 BT 下载的场景中可能会受到影响。另外，Dropbox 会提供有文件版本存档，有需要时可以将文件恢复到之前的版本，而 Resilio Sync 目前还没有这种功能。

软件使用

这个软件引入了“同步密钥”的概念

每个参与同步的目录，都有其密钥。你只有拿到这个密钥，才能同步该目录的文件。

对于普通的使用场景，每个同步目录对应两个密钥：一个是“读写密钥”，另一个是“只读密钥”。顾名思义，拥有“读写密钥”的节点，可以修改同步目录的内容；反之，拥有“只读密钥”的节点，只能读取，无法修改——所谓的“无法修改”，就是说：即使你修改了同步目录的内容，修改结果也不会同步给其它节点（所以这种修改是无意义的）。

密钥长度应该是32位字符，即使暴力破解，估计也得很久，久到让人不觉得值得破解……官网虽然是英文的，不过软件现在安装后就会是中文的，并不难使用。
